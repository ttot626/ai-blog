## 二、服务器部署与更新（每次改代码后）

### 第一次部署（已完成可跳过）

```bash
git clone https://github.com/ttot626/ai-blog.git
cd ai-blog
cp .env.example .env
nano .env          # 改 DB_PASSWORD、JWT_SECRET、DEEPSEEK_API_KEY
docker compose up -d --build
```

### 日常更新（最常用）

```bash
cd ~/ai-blog
git pull
docker compose up -d --build
```

**说明：** 首次或大量改动后 `build` 可能要 **20～30 分钟**（2 核 2G 机器正常），Maven 编译时屏幕可能长时间无输出，耐心等。

### 只改了 .env（没改代码）

```bash
docker compose up -d --force-recreate app
```

`restart` **不会**重新读 `.env`，必须用 `force-recreate`。

### 部署成功怎么验

```bash
docker compose ps                              # 三个容器都是 running / healthy
docker compose logs app --tail 20              # 看到 Started Xiangmu1Application
curl http://localhost:8080/hello               # 返回 code 200
curl "http://localhost:8080/article/list?page=1&size=10"   # 有 records、total、page
```

浏览器打开 http://47.97.85.200:8080 能注册、登录、发文即可。

## 六、面试只背 5 个问题（够用了）

### 1. JWT 怎么用？
登录成功返回 Token；之后请求 Header 带 `Authorization: Bearer xxx`；拦截器解析出 userId 放进 ThreadLocal。

### 2. Redis 缓存了什么？
文章分页列表、热门榜、用户主页。发文/删文/点赞后会清缓存，避免读到旧数据。

### 3. 密码怎么存？
BCrypt 加密，库里不存明文；老账号首次登录会自动升级成 BCrypt。

### 4. 分页怎么做？
Controller 收 `page`、`size` → Service 用 MyBatis-Plus 的 `Page` 查库 → 返回 `PageResult`（records、total、pages）。

### 5. 怎么部署的？
Docker Compose 起 MySQL + Redis + Spring Boot；`.env` 配密码和密钥；安全组放行 8080。

## 八、常见问题

| 现象 | 处理 |
|------|------|
| 外网打不开 | 阿里云安全组放行 **8080** |
| Docker build 很久 | 2G 机器正常，等 20～30 分钟；可加 swap |
| build 被 Killed | 内存不够，`fallocate -l 2G /swapfile && mkswap && swapon` 后重试 |
| AI 报未配置 Key | 检查 `.env`，然后 `force-recreate app` |
| AI 402 余额不足 | DeepSeek 控制台充值 |
| 改 .env 不生效 | 必须 `docker compose up -d --force-recreate app` |
