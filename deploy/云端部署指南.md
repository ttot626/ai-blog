# AI Blog 云端部署指南

让**所有人**通过同一个网址在线注册、发博客、看博客。

---

## 两种用法对比

| | GitHub 下载 zip（桌面版） | 云端部署（本指南） |
|---|---|---|
| **谁在用** | 每个人自己电脑各跑一份 | 所有人共用一个网站 |
| **数据** | 各自 MySQL，互不相通 | 同一数据库，文章全员可见 |
| **访问** | 只能本机 `localhost:8080` | `http://你的域名或IP:8080` |
| **适合** | 本地演示、作业、离线使用 | 公开博客、多人社区 |

---

## 方案：Docker 一键部署（推荐）

需要一台 **Linux 云服务器**（阿里云 / 腾讯云 / 华为云均可）。

**最低配置：** 2 核 CPU · 2GB 内存 · 40GB 硬盘 · Ubuntu 22.04

---

### 第 1 步：购买云服务器

1. 注册阿里云或腾讯云
2. 购买 **轻量应用服务器** 或 **ECS**（选 Ubuntu 22.04）
3. 记下 **公网 IP**
4. 在安全组 / 防火墙中 **放行 8080 端口**（若用域名 + Nginx 则放行 80、443）

---

### 第 2 步：安装 Docker

SSH 登录服务器后执行：

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# 重新登录 SSH 使 docker 组生效
```

验证：

```bash
docker --version
docker compose version
```

---

### 第 3 步：拉取代码并配置

```bash
git clone https://github.com/ttot626/ai-blog.git
cd ai-blog
cp .env.example .env
nano .env   # 或用 vi 编辑
```

**必须修改 `.env`：**

```env
DB_PASSWORD=你的强密码
JWT_SECRET=随机长字符串至少32位
DEEPSEEK_API_KEY=你的密钥（可选）
```

---

### 第 4 步：启动

```bash
docker compose up -d --build
```

首次启动约 3～5 分钟（下载镜像 + 编译 + 初始化数据库）。

查看状态：

```bash
docker compose ps
docker compose logs -f app
```

看到 `Started Xiangmu1Application` 即成功。

---

### 第 5 步：访问

浏览器打开：

```text
http://你的公网IP:8080
```

注册账号 → 登录 → 发文章，**所有访问该地址的人都能看到**。

接口文档：`http://你的公网IP:8080/swagger-ui.html`

---

## 绑定域名（可选）

若有域名 `blog.example.com`：

1. DNS 添加 A 记录指向服务器 IP
2. 安装 Nginx 反向代理：

```nginx
server {
    listen 80;
    server_name blog.example.com;
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

3. 用 Certbot 申请免费 HTTPS 证书

## 常用运维命令

```bash
# 停止
docker compose down

# 重启
docker compose restart app

# 更新代码后重新部署
git pull
docker compose up -d --build

# 查看日志
docker compose logs -f app
```

## 安全提醒

- 生产环境务必修改 `DB_PASSWORD` 和 `JWT_SECRET`
- 不要将 `.env` 提交到 Git
- 建议后续将用户密码改为 BCrypt 加密

---

## 故障排查

| 现象 | 处理 |
|------|------|
| 外网打不开 | 检查云厂商安全组是否放行 8080 |
| 启动报错数据库 | `docker compose logs mysql` 查看初始化 |
| 502 / 连接拒绝 | `docker compose ps` 确认 app 在运行 |
| 表不存在 | `docker compose down -v` 后重新 `up`（会清空数据） |
